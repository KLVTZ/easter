~ (Object) backoff(fn, options = {})

This module implements a backoff mechanism when calling the given function in a recursive manner. Ie. the call to fn() is done repeteadly, whenever the promise returned is resolved the interval shrinks to a minimum low, whenever the promise is rejected the interval increases to a maximum high.

Parameters:

  • fn ( Function ) —
      a function that MUST return a promise.
  • options ( Object )

Options Hash: (strategy):

  • One ( String ) — of 'fibonacci' or 'exponential'.

Options Hash: (randomisationFactor):

  • increasing ( Number ) — call intervals can be made more variable using this option. Can be from [0, 1]. Defaults to 0

Options Hash: (initialDelay):

  • how ( Number ) — much to wait in ms to get the first call going. Defaults to 100ms.

Options Hash: (maxDelay):

  • the ( Number ) — maximum delay between calls.

Options Hash: (startImmediately):

  • whether ( Boolean ) — or not to start the backoff process immediately. Defaults to false.

Returns:

  • ( Object ) — instance of backoff.Backoff class.

See also:

  • https://github.com/MathieuTurcotte/node-backoff

~ (void) pipe(steps, context = {}, options = {})

Note: Normal behaviour is to stop to the pipe.

Utility method that executed all steps given and applies the context object resulted form the previous step to the next step.

Parameters:

  • steps ( Array<Function> ) — A list of functions, each one receives the context object as param, modify it, then pass it onto the next step.
  • context ( Object ) — An object which passes through the step pipe.
  • options ( Object )

Options Hash: (ignoreErrors):

  • If ( Boolean ) — this flag is true, if a step in the pipeline errors, the next step will execute.
