# (Q.Promise) queue(queueName, queueOpts = {})

Method defines/creates a queue. Calls to RabbitMQ are idempotent so the queue will not be created twice. However we are caching the queue promise to prevent unnecessary requests.

Parameters:

  • queueName ( String ) — The queue will be created if it does not exist.
  • queueOpts ( Object ) — queue specific options.

Options Hash: (passive):

  • Server ( Boolean ) — will attempt to create the queue if it does not exist yet. Defaults to false.

Options Hash: (autoDelete):

  • Makes ( Boolean ) — the queue to self-destruct when all consumers have finished with it, ie. close their channel to the queue. Defaults to false.

Options Hash: (durable):

  • Durable ( Boolean ) — queues are persistent, ie support persistent messages. All non-persistent messages will be lost in case of a reboot. Defaults to true.

Options Hash: (exclusive):

  • Exclusive ( Boolean ) — queues can be consumed only from the current location. Defaults to false.

Options Hash: (noDeclare):

  • Allow ( Boolean ) — the queue to be deleted even if the client does not know it's options. By default this is not permitted. Defaults to false.

Returns:

  • ( Q.Promise ) — resolving to a Queue instance.

# (void) subscribe(queueName, callback, options = {})

Note: Call ack() to consume the message, or it will get republished!

Subscribe to a rabbitmq queue. Whenever a message is received, the callback is executed with that message.

Local function is used by event callbacks to trigger the ACK for the queue. If an error is passed, it's logged but the message is considered to be consumed.

Parameters:

  • queueName ( String ) — name of the queue to subscribe to.
  • callback ( Function ) — continuation, signature is function (error, data, ack) {}
  • options ( Object ) — Options specific to the messages consumed.

Options Hash: (ack):

  • Makes ( Boolean ) — Rabbit only send a new message to the consumer when the ACK from the last one was received. To send ACK, use queue#shift(). Defaults to true.

Options Hash: (prefetchCount):

  • Number ( Boolean ) — of messages allows to wait in the worker's buffer at a time. Defaults to 1.
